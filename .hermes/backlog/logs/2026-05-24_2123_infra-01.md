Tâche: Configurer UFW firewall sur 91.99.26.43. Autoriser (22, 80, 443, 1883) et bloquer tout le reste.
Résultat: ❌ ÉCHEC
---------------------------------------------------
Commandes tentées (sur VPS 91.99.26.43):
1. sudo ufw default deny incoming
2. sudo ufw allow 22/tcp
3. sudo ufw allow 80/tcp
4. sudo ufw allow 443/tcp
5. sudo ufw allow 1883/tcp
6. echo "y" | sudo ufw enable
7. sudo ufw status verbose

ERREUR (toutes les commandes):
sudo: A terminal is required to authenticate

Cause: Le mot de passe sudo est requis sur le VPS. L'utilisateur noisyless@91.99.26.43 n'a pas de configuration NOPASSWD dans /etc/sudoers.

Action requise: Mathieu doit soit fournir le mot de passe sudo, soit configurer NOPASSWD pour UFW (sudo visudo → ajouter "noisyless ALL=(ALL) NOPASSWD: /usr/sbin/ufw").
