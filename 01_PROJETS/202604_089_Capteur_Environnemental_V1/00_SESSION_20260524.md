## 3. ARCHITECTURE MÉMOIRE

### Capteur → Cloud

```
ESP32-S3 (192.168.0.176 en dev)
  │
  ├─ WiFi → MQTT Broker (91.99.26.43:1883)
  │           │
  │           └─ mqtt-ingest → TimescaleDB (heatmap_agg_5min)

```

### Capteurs embarqués

| Capteur | Interface | Rôle |
|---------|-----------|------|
| BME680 + BSEC2 | I2C (pins 35/36) | Qualité d'air (IAQ, CO₂eq, bVOC, T°, HR%, pression) |
| LD2410 | UART (pins 17/18) | Radar présence + distance |
| 2× LDR | ADC (pins 1/2) | Luminosité |
| 2× Micros | ADC (pins 3/4) | Audio (pic-à-pic) |

## 8. RISQUES & GARDE-FOUS

| Risque | Mitigation |
|--------|------------|
| mqtt-ingest écrit dans mauvaise table | ✅ Identifié, à patcher |
| Firmware trop gros (92.8% flash) | ⚠️ Marge faible (72KB restants) — optimiser si needed |
| OTA hash non calculé | ⚠️ Mettre `sha256:pending_build` → calculer après build |
| WiFi ne se connecte pas | Vérifier credentials WiFiManager, timeout 5s |
| BSEC2 state non persisté | Prévoir RTC memory pour deep sleep (futur) |
